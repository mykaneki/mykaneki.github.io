rdt演变>

rdt演变 #

  • rdt 1.0
    • 完全可靠信道
  • rdt 2.0
    • 具有比特差错，假设ACK和NAK无损
    • 使用ACK和NAK建立自动重传请求ARQ
    • 停等协议：每发送一个分组都要接受端的确认信号
  • rdt 2.1
    • 解决rdt 2.0 信号受损
    • 给分组设置序号
  • rdt 2.2
    • 只有ACK没有NAK
  • rdt 3.0
    • 比特差错和分组丢失
    • 超时重传
  • rdt 3.0 改良
    • rdt3.0性能太低
    • 引入流水线协议
      • 差错恢复（GBN\SR）
